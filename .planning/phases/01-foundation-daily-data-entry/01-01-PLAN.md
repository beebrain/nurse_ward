<objective>
Initialize the CodeIgniter 4 environment and set up the foundational authentication system using Shield.

Purpose: To establish a secure, framework-compliant base for all subsequent features.
Output: A working CI4 application with Shield Auth installed and session security configured.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Initialize CI4 and Install Shield Auth</name>
  <files>composer.json, .env, app/Config/Auth.php</files>
  <action>
    - Initialize CodeIgniter 4 using composer (appstarter).
    - Install `codeigniter4/shield`.
    - Set up the `.env` file with database credentials (per PROJECT.md).
    - Run `php spark auth:setup` to initialize Shield tables and configuration.
  </action>
  <verify>
    <automated>php spark auth:list_users</automated>
  </verify>
  <done>CI4 is installed and Shield tables exist in the database.</done>
</task>

<task type="auto">
  <name>Task 2: Configure Session Security and RBAC Foundations</name>
  <files>app/Config/Security.php, app/Config/App.php, app/Config/AuthGroups.php</files>
  <action>
    - Set `sessionExpiration` to 7200 (2 hours) in `app/Config/App.php` per D-03 (01-CONTEXT.md).
    - Enable CSRF protection in `app/Config/Security.php`.
    - Define 'superadmin', 'manager', and 'nurse' groups in `app/Config/AuthGroups.php`.
  </action>
  <verify>
    <automated>grep -E "sessionExpiration.*7200" app/Config/App.php</automated>
  </verify>
  <done>Session timeout is set to 2 hours and roles are defined.</done>
</task>

</tasks>

<verification>
Check that the base URL loads and the `/login` route provided by Shield is accessible.
</verification>

<success_criteria>
- CI4 framework is responsive.
- Shield authentication tables are present.
- User session expires after 2 hours of inactivity.
</success_criteria>
